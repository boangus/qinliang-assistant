## 1. 项目背景

### 1.1 竞赛信息

- **赛事**: TRAE AI 创造力大赛
- **赛道**: 社会服务
- **阶段**: 初赛（2026.6.16 — 7.15）
- **要求**: 用 TRAE IDE/Work 开发可体验 Demo，发布到社区初赛专区
- **评分维度**: 创意想法(25%) + 产品设计(25%) + 技术实现(25%) + 展示效果(25%)

### 1.2 产品定位

「沁凉助手」是一个面向**潮热盗汗患者**的 AI 智能管理工具，覆盖前列腺癌 ADT 患者和更年期女性两大人群。核心功能包括症状记录、AI 预测、数据可视化、健康科普、中医耳穴指导、体质评估和科研数据采集。

### 1.3 现有基础

已有微信小程序版本（`boangus/WechatProgram`），包含 26 个页面、8 个云函数、三角色架构。本次比赛需要将其核心功能复刻为浏览器可体验的 HTML Demo。

## 2. 架构设计

### 2.1 技术选型

| 组件 | 选择 | 用途 |
|------|------|------|
| UI 框架 | Vue 3 CDN (3.5.x) | 响应式数据绑定、组件化 |
| 样式 | TailwindCSS CDN (3.x) | 原子化 CSS |
| 图表 | ECharts CDN (5.x) | 趋势图、饼图、热力图 |
| Markdown | marked.js CDN | 科普文章渲染 |
| 数据持久化 | localStorage | 模拟云数据库 |

**离线策略**: Zip 包内包含所有 CDN 库的本地副本，HTML 文件引用本地路径，确保离线可运行。

### 2.2 导航结构

模拟微信小程序 TabBar 的底部导航：

```
┌──────────────────────────────────┐
│          [页面内容区域]            │
│                                  │
│                                  │
│                                  │
├──────────────────────────────────┤
│  🏠首页  │ 📝记录 │ 📖科普 │ 👤我的  │
└──────────────────────────────────┘
```

- 4 个 Tab 页：首页(Dashboard)、记录(Diary)、科普(Education)、我的(Profile)
- 子页面通过 hash 路由实现（`#/stats`、`#/auricular`、`#/constitution`、`#/scale`、`#/vms-survey`、`#/treatment` 等）

### 2.5 预设演示数据

首次加载时自动生成 30 天模拟数据，确保评审打开即可看到完整效果：

- **30 天 × 平均 4 次/天** = ~120 条发作记录
- 严重度分布：轻度 35%、中度 40%、重度 20%、极重度 5%
- 高发时段：14:00-16:00（午后）、02:00-04:00（夜间）
- 常见诱因：热咖啡、辛辣食物、高温环境
- 诱因调整后症状趋势下降（展示工具价值）

## 3. 功能模块详细设计

### 3.1 首页仪表盘 (Dashboard)

**布局**: 移动端优先，单列卡片流

**组件**:
1. **今日概览卡片**
   - 今日潮热次数（大数字 + 较昨日对比箭头）
   - 严重度分布标签
   - 连续记录天数

2. **本周趋势图** (ECharts 折线图)
   - X 轴：近 7 天日期
   - 双 Y 轴：左-发作次数（柱状），右-平均严重度（折线）
   - 标注高发日

3. **AI 建议卡片**
   - 时段预警："下午 14:00-16:00 为你的潮热高发时段"
   - 诱因提醒："热咖啡是你的主要诱因之一"
   - 干预推荐："建议尝试 4-7-8 呼吸法"
   - 每日科普冷知识

4. **快速记录浮层**
   - 点击按钮弹出简化记录表单
   - 严重度选择 + 时间默认当前 + 保存

### 3.2 症状记录 (Diary)

**记录表单字段**:

| 字段 | 类型 | 选项 |
|------|------|------|
| 时间 | datetime-local | 默认当前时间 |
| 严重度 | radio | 轻度/中度/重度/极重度 |
| 持续时间 | number (分钟) | 滑块 1-60 |
| 诱因 | checkbox | 咖啡因/酒精/辛辣食物/高温环境/压力/剧烈运动/情绪波动/其他 |
| 伴随症状 | checkbox | 面部潮红/心悸/出汗/失眠/焦虑/头痛/其他 |
| 干预措施 | select | 无/腹式呼吸/耳穴按压/穴位按摩/药物/其他 |
| 备注 | textarea | 自由文本 |

**日记列表**:
- 按日期分组显示
- 每条记录显示：时间、严重度彩色标签、主要诱因
- 支持左滑删除、点击编辑

### 3.3 数据统计 (Stats)

**图表模块**:

1. **频率趋势图** (ECharts)
   - 日/周/月 三个视图切换
   - 折线图 + 面积填充

2. **严重度分布** (ECharts 饼图)
   - 4 级严重度占比
   - 中心显示总数

3. **诱因关联分析** (ECharts 水平柱状图)
   - 各诱因出现频次
   - 按频次降序排列

4. **时间段热力图** (ECharts 热力图)
   - 24 小时 × 7 天矩阵
   - 颜色深浅表示发作频次

5. **干预效果对比** (ECharts 分组柱状图)
   - 有干预 vs 无干预的平均严重度对比

### 3.4 AI 潮热预测引擎

**非 ML 规则引擎，展示 AI 概念**:

1. **时间模式分析**
   - 统计各小时的历史发作概率
   - 识别 Top-3 高发时段

2. **诱因关联分析**
   - 计算各诱因后的发作概率提升
   - 生成个性化避忌建议

3. **预测输出**
   - 未来 24 小时时段风险评估（高/中/低）
   - "下午 14:00-16:00 潮热概率: 78%（高风险）"
   - 建议提前采取预防措施

4. **准确率回测**
   - 用历史数据验证：将过去 N 天的数据按时间模式预测，计算命中率
   - 展示 "过去 7 天预测准确率: 72%"

5. **个性化建议生成**
   - 基于诱因统计 → "减少咖啡因摄入可能降低 23% 的发作频率"
   - 基于时间模式 → "睡前进行 5 分钟腹式呼吸可能减少夜间发作"
   - 基于干预效果 → "耳穴按压后发作严重度平均降低 1.2 级"

### 3.5 健康科普 (Education)

**文章分类与内容**:

| 分类 | 文章数 | 内容示例 |
|------|--------|---------|
| 潮热机制 | 2 | 潮热的生理机制 / 为什么 ADT 会引发潮热 |
| 诱因管理 | 3 | 饮食与潮热 / 环境温度管理 / 压力与潮热 |
| 中医调理 | 3 | 耳穴压豆原理 / 穴位按摩指南 / 中医体质调理 |
| 西医药物 | 2 | HRT 治疗简介 / 新型 NK3R 拮抗剂 |
| 生活技巧 | 3 | 分层穿衣法 / 4-7-8 呼吸法 / 睡眠卫生 |

**功能**:
- 分类 Tab 切换
- 文章卡片列表（标题 + 摘要 + 分类标签）
- 文章详情：Markdown 渲染，支持标题、段落、列表、加粗、图片
- 每日冷知识卡片（首页展示）

### 3.6 耳穴压豆指南 (Auricular)

**10 个标准穴位**:

| 穴位 | 定位 | 主治 |
|------|------|------|
| 神门 | 三角窝后 1/3 上部 | 镇静安神、调节自主神经 |
| 交感 | 耳轮下脚末端 | 调节交感/副交感神经平衡 |
| 内分泌 | 屏间切迹底部 | 调节内分泌功能 |
| 肾 | 对耳屏下方 | 补肾固本 |
| 肝 | 耳甲艇后下方 | 疏肝理气 |
| 皮质下 | 对耳屏内侧面 | 调节大脑皮层功能 |
| 心 | 耳甲腔中央 | 宁心安神 |
| 脾 | 耳甲腔上方 | 健脾益气 |
| 肺 | 耳甲腔中央周围 | 宣肺理气 |
| 三焦 | 耳甲腔底部 | 通调水道 |

**功能**:
- 穴位列表（卡片式，含穴位名称 + 一句话定位）
- 穴位详情页（定位方法图文 + 按压技巧 + 主治说明）
- **按压计时器**：选择穴位 → 开始 → 30 秒倒计时动画 → 完成提示

### 3.7 中医体质评估 (Constitution)

**9 种体质 × 8 题 = 72 题**（硬编码）

- 平和质（8 题，部分反向计分）
- 气虚质 / 阳虚质 / 阴虚质 / 痰湿质 / 湿热质 / 血瘀质 / 气郁质 / 特禀质（各 8 题）

**计分规则**: 每题 1-5 分，计算各体质维度得分，取最高分为主要体质。

**结果展示**: 体质类型 + 得分雷达图（ECharts） + 调理建议

### 3.8 通用量表引擎 (Scale)

**从内嵌 JSON 配置加载量表**:

- 支持多类别（category）结构
- 支持正向/反向计分
- 支持 `sum`、`percentage`、`category_sum` 三种计分规则
- 结果模板：根据得分区间返回等级和建议

**预置量表**: 1 个示例量表（可由管理员扩展）

### 3.9 VMS 研究问卷 (VmsSurvey)

**9 步问卷流程**:

1. 基本信息（年龄、诊断、治疗方案）
2. 患者信念评估
3. VMS 频率评估
4. HFNS（Hot Flash Night Score）
5. HFRDIS（Hot Flash Related Daily Interference Scale）
6. HFBBS-Men（Hot Flash Beliefs and Behaviors Scale）
7. 治疗体验
8. 研究指标
9. 满意度

**标准量表**: HFNS、HFRDIS、HFBBS-Men 均按原始文献的计分规则实现。

### 3.10 分级治疗推荐 (Treatment)

**三级分层**:

| 层级 | 类型 | 内容 |
|------|------|------|
| 第一层 | 生活方式 | 4-7-8 呼吸法、腹式呼吸、环境温控、饮食避忌、运动建议 |
| 第二层 | 中医非药物 | 耳穴压豆（配穴方案）、穴位按摩（三阴交/太溪/涌泉）、放松训练 |
| 第三层 | 药物参考 | HRT 简介、SSRI/SNRI、加巴喷丁、NK3R 拮抗剂（科普，非处方） |

每项干预附：操作步骤、注意事项、循证证据等级标注。

### 3.11 个人中心 (Profile)

- 用户信息编辑（姓名、性别、年龄、病情）
- 体质评估入口
- 通用量表入口
- VMS 问卷入口
- 数据管理（清除数据、导出为 JSON）
- 关于页面（产品介绍、版本信息）

## 7. 约束与假设

1. **单文件约束**: 所有功能在单个 HTML 文件中实现，CSS/JS 内联或通过本地文件引用
2. **离线可用**: Zip 包内含所有依赖库，无需网络即可运行
3. **演示数据**: 预设 30 天模拟数据，打开即可体验
4. **无后端**: 所有数据存储在 localStorage，无云函数
5. **非 ML**: AI 预测基于规则引擎（统计模式分析），非机器学习模型
6. **中文优先**: 界面全中文，适合国内评审
7. **移动端模拟**: 页面以 375px 宽度居中展示，模拟手机体验

## 8. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 单 HTML 文件过大 | 加载慢 | 代码压缩 + 按需加载图表 |
| localStorage 容量限制 | 数据量受限 | 控制演示数据量在 5MB 以内 |
| ECharts 体积大 | Zip 包过大 | 使用 echarts.min.js (~1MB) |
| 功能过多时间不够 | 无法按时完成 | 优先核心 5 模块，其余按优先级递减 |
